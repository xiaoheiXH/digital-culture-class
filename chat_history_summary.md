# 校园地图 H5 项目：对话交互记录摘要
**日期**：2026-04-12
**存档说明**：本文件记录了在开发过程中，用户与 AI 助手之间的关键对话、需求变更以及技术攻关过程。

---

## 1. 初始阶段：功能对齐
- **用户需求**：询问是否能写 Word 文档（用于实验报告），随后转向 H5 搜索框功能的实装。
- **技术动作**：AI 检查了 `app.js`，发现搜索框仅为样式占位，随后实现了实时的地点过滤和结果展示列表。

## 2. 核心演进：从本地到云端
- **用户构思**：提出将项目部署在 GitHub 和 Vercel 上，实现“永久且简便”的线上共享。
- **技术决策**：确定使用 Vercel KV (Redis) 作为轻量级数据库。AI 为项目增加了 `package.json` 和 `api/points.js` 后端接口文件。

## 3. 架构深化：建筑图层与权限
- **用户想法**：希望在地图上叠加建筑描绘图，并让管理员能配置详细信息。
- **技术动作**：AI 预留了 `ImageLayer`（图片叠加层）和 `Polygon`（建筑轮廓感应区）的代码，并将图层暂时设为透明以保持界面整洁。
- **权限管理**：根据用户要求，实装了登录界面，内置管理员账号 `bonvoyage`。设定了“所有人可新建标点，仅管理员可改删”的权限逻辑。

## 4. 攻坚阶段：同步故障排查
- **挑战 A**：用户发现云端同步显示“获取 0 个地点”。
  - **诊断**：云端为空时，代码未自动初始化。
  - **解决**：增加了“自动初始化”逻辑，当云端为空时自动推送本地数据。
- **挑战 B**：浏览器报出 `QuotaExceededError`（存储空间爆满）。
  - **原因**：图片 Base64 字符串过大。
  - **解决**：重构了保存逻辑，使用 `try...catch` 确保即使本地存不下，也必须优先同步至云端。

## 5. 交互优化：评论与 UI
- **用户需求**：要求匿名用户也能填名字发表评论，并希望将“同步”按钮隐藏在设置中。
- **技术动作**：升级了评论区 UI，增加了昵称输入框；创建了“系统设置”弹窗，将高级功能归位。

---

## 6. 用户关键语录与决策
- “如果我在 GitHub 上上架这个库然后在 Vercel 上架设前端可以永久个最简便的方法实现可能性吗” —— **确立了云端化技术路线**。
- “我需要所有人都可以匿名使用（甚至可以暂时标点）只是我的界面可以登录” —— **确立了开放式 UGC 权限设计**。
- “本地启动倒是同步失败...但是在此之前我确确实实标点了个新的地标” —— **触发了云端同步逻辑的深度优化**。
